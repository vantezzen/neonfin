---
title: Connect a provider
description: Connect Stripe or Polar so vantezzen/pay can charge for credits.
---

vantezzen/pay never holds money. A payment provider - **Stripe** or **Polar** - owns
checkout, payment methods, invoices, taxes, and subscription billing. vantezzen/pay
owns products, credits, credit codes, and webhook fulfillment. This step is the
same whether you use the hosted instance at `pay.vantezzen.io` or self-host.

Connect a provider from the dashboard under **Providers**. vantezzen/pay creates and
syncs the product catalog inside your provider account for you - you never
manage the catalog in the provider dashboard.

For step-by-step Stripe and Polar setup, including the current Stripe Developer
Workbench webhook flow, see [Connect a payment provider](/docs/workflows/providers).
For a side-by-side list of supported provider behavior, see the
[Provider capability matrix](/docs/workflows/provider-capabilities).

<Callout>
Use a **dedicated** Stripe or Polar account for vantezzen/pay-managed products if you
can. vantezzen/pay creates catalog objects automatically, and a dedicated account
keeps them clearly separate from anything you manage by hand.
</Callout>

## Stripe

1. In **Providers → Connect provider**, choose Stripe and paste an API key.
   Use `sk_test_…` while developing; use a live key from the account that should
   own the products and prices in production.
2. Add the webhook endpoint vantezzen/pay shows you. Subscribe to:
   - `checkout.session.completed`
   - `invoice.paid`
   - `charge.refunded`
   - `customer.subscription.deleted`
3. Paste the webhook signing secret back into vantezzen/pay.

For local development, forward events with the Stripe CLI - vantezzen/pay shows the
exact command, e.g.:

```bash
stripe listen --forward-to https://pay.example.com/api/webhooks/stripe/acct_123
```

## Polar

1. In **Providers → Connect provider**, choose Polar and paste an organization
   access token that can manage catalog, checkout, customer sessions, and
   webhooks.
2. Add the webhook endpoint vantezzen/pay shows you. Subscribe to:
   - `order.paid`
   - `order.refunded`
   - `subscription.canceled`
   - `subscription.revoked`
3. Paste the webhook signing secret back into vantezzen/pay.

## Test before going live

- Create a product and price in vantezzen/pay.
- Start checkout from a test app.
- Complete payment in the provider's test mode.
- Confirm the order becomes **paid** in vantezzen/pay.
- Confirm credits appear in the wallet.
- Trigger a refund and confirm the webhook log records it.
