This guide follows the easiest setup:

- Stripe as the payment provider.
- Credit codes instead of user accounts.
- One product.
- One price.
- One paid feature in your app.

That is the path most side projects should try first. Polar and existing-auth
setups are supported too, but they are easier to understand after this flow.

<Callout title="Setting up with an AI agent?">
If a coding agent (Claude Code, Cursor, Copilot, …) is doing the integration, just use this prompt:

```text
Integrate vantezzen/pay into this app from start to finish.
For that, please read the guide at https://pay.vantezzen.io/docs/agent.mdx
```
</Callout>

## Before you start

You need:

- A vantezzen/pay instance, either hosted or [self-hosted](/docs/self-host).
- A Stripe account in test mode.
- A React app using Tailwind and shadcn.

<Callout>
If your app is not using shadcn yet, run `npx shadcn@latest init` in that app
before installing vantezzen/pay components.
</Callout>

## The path

1. [Set up vantezzen/pay and Stripe](/docs/getting-started/dashboard).
2. [Install the SDK and components](/docs/getting-started/install).
3. [Gate your first feature](/docs/getting-started/first-feature).
4. [Check the go-live list](/docs/getting-started/go-live).

By the end, your app will have working anonymous credit wallets. Users do not
need to create an account unless your app already has authentication and you
choose external auth mode.
